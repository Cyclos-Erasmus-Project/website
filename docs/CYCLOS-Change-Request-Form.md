## Instructions for Stakeholders

This form is used to collect information needed to create a formal change proposal for the CYCLOS website. Please complete all sections as thoroughly as possible. The information you provide will be used by Computer Solutions SA to create a technical specification that guides the implementation.

### How to Submit

1. **Download** this document or copy the questionnaire section
2. **Complete** all required fields (marked with *)
3. **Send** the completed form to Computer Solutions SA via email
4. **Review** the technical proposal when returned for your approval
5. **Approve** before implementation begins

### Change Request Workflow


## Appendix A: Change Type Decision Tree

Use this tree to determine if your request requires a formal change proposal:

```
Is this a change to the CYCLOS website?
│
├─ YES
│   │
│   ├─ Is it fixing a typo or formatting error?
│   │   └─ NO proposal needed - email correction directly
│   │
│   ├─ Is it updating existing content (dates, text, translations)?
│   │   └─ SIMPLE proposal - complete Parts A, B, C, D
│   │
│   ├─ Is it adding new functionality or pages?
│   │   └─ FULL proposal - complete all parts
│   │
│   ├─ Is it changing design, layout, or navigation?
│   │   └─ FULL proposal - complete all parts
│   │
│   └─ Is it a bug that breaks intended behavior?
│       └─ NO proposal needed - email bug report directly
│
└─ NO
    └─ This form is not applicable
```

## Appendix B: Example Completed Request

Below is an example of a properly completed change request for reference.

---

### Example: Add Project Event Calendar

**Part A: Requestor Information**

| Field | Response |
|-------|----------|
| Full Name | Marie Dupont |
| Organization | FLVS (France) |
| Email | m.dupont@flvs.fr |
| Date of Request | 15 January 2026 |
| Work Package Reference | WP5 |

**Part B: Change Identification**

| Field | Response |
|-------|----------|
| Change Title | Add project event calendar to homepage |
| Change Type | New Feature |
| Priority | Medium |
| Target Date | 28 February 2026 (before first transnational meeting) |

**Part C: Change Description**

**C1. Why is this change needed?**
```
Partners need visibility into upcoming project events to coordinate attendance 
and travel arrangements. Currently, event information is only shared via email, 
which leads to missed communications. A public calendar on the website will 
improve coordination and demonstrate project activity to external stakeholders.
```

**C2. What specifically should change?**
```
- Add a new "Upcoming Events" section to the homepage
- Display next 3-5 events with date, location, and brief description
- Add a "View All Events" link to a dedicated events page (future)
- Include event type indicators (meeting, workshop, dissemination)
```

**C3. Which pages are affected?**
- [x] Homepage
- [ ] Other pages unaffected

**C4. Languages affected?**
- [x] All 5 languages (EN, FR, RO, DE, EL)
- Translation: Provided for FR, needs translation for others

**Part E: Success Criteria**

```
Scenario 1:
- WHEN: A user visits the CYCLOS homepage
- THEN: They see an "Upcoming Events" section showing the next 3 events

Scenario 2:
- WHEN: A user views an event in the list
- THEN: They see the event date, location, type, and brief description

Scenario 3:
- WHEN: There are no upcoming events
- THEN: The section displays "No upcoming events scheduled"
```

## Appendix C: Glossary

| Term | Definition |
|------|------------|
| **Change Proposal** | A formal document describing a requested change to the website |
| **Spec/Specification** | A document defining the requirements and expected behavior |
| **Delta** | The difference between current state and proposed state |
| **WCAG** | Web Content Accessibility Guidelines - accessibility standards |
| **Scenario** | A specific use case describing expected behavior |
| **Breaking Change** | A change that modifies existing functionality |
